## 🎨 自定义字体

问题：自定义字体不生效

### 现象

- 字体文件已添加但显示为系统字体
- 控制台输出字体加载失败警告
- 不同 iOS 版本表现不一致

### 解决方案

#### 1. 配置 Info.plist

```xml
<!-- Info.plist -->
<key>UIAppFonts</key>
<array>
    <string>BebasNeue-Regular.ttf</string>
    <string>PingFang-SC-Medium.ttf</string>
    <!-- 注意：这里填写的是文件名，不是字体家族名 -->
</array>
```
#### 2. 正确加载字体

```swift
UIFont(name: "BebasNeue-Regular", size: 12)
```

### 自定义字体问题排查

#### 1. 文件检查
- 字体文件已添加到项目中
- 文件在 Copy Bundle Resources 中
- 文件名拼写正确（包括扩展名）

#### 2. 配置检查  
- Info.plist 中 UIAppFonts 配置正确
- 数组中的文件名与实际文件一致
- 没有多余的空格或特殊字符

#### 3. 代码检查
- 使用正确的字体家族名（不是文件名）
- 字体名称大小写敏感
- 添加了加载失败的回退方案

#### 4. 运行时检查
- 打印所有可用字体确认加载成功
- 检查控制台错误信息
- 在不同设备上测试

## 国际化与本地化

### [InfoPlist.strings](./KindsOf_Swift/InfoPlist.strings)

InfoPlist.strings 是 iOS/macOS 等苹果平台的本地化字符串文件，核心作用是为 Info.plist 中的关键配置（如隐私权限描述、App 名称、功能说明等）提供多语言支持，避免硬编码文本，同时满足 App Store 审核对「权限说明透明度」的要求（尤其隐私相关描述是必配项）
