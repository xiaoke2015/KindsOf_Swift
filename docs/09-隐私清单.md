#  09-隐私清单

## 背景

### [提醒：适用于 App 提交的隐私要求将从 5 月 1 日开始实施](https://developer.apple.com/cn/news/?id=pvszzano)
2024 年 4 月 26 日 App Store 公告，5 月 1 日起，使用常用列表中新增第三方 SDK 的新 App 或更新 App 提交时，须有列出 API 的使用原因、隐私清单及对应有效签名，不达标将不被批准。未来该声明要求还将扩大至整个 App 二进制文件，以此保障用户隐私。

## 配置指南

### 1.基础文件结构
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>NSPrivacyAccessedAPITypes</key>
    <array>
        <!-- 访问的API类型 -->
    </array>
    <key>NSPrivacyCollectedDataTypes</key>
    <array>
        <!-- 收集的数据类型 -->
    </array>
    <key>NSPrivacyTracking</key>
    <true/>
    <!-- 或 <false/> -->
</dict>
</plist>
```

### 2.文件位置要求

```
Project/
├── PrivacyInfo.xcprivacy ✅
├── Info.plist
└── Sources/
```

## 第三方SDK集成

### 现象

- 集成第三方SDK后审核被拒
- SDK提供商未提供隐私清单文件
- 多个隐私清单文件冲突

### 解决方案
查看官方要求的三方 sdk 需要配置 更新Sdk
https://developer.apple.com/cn/support/third-party-SDK-requirements/
